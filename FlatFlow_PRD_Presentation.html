Problem & Solution

Why FlatFlow Exists

🔴 The Problem

  • Housing societies rely on manual registers, WhatsApp groups & paper receipts
  • No transparency in complaint tracking — residents never know status
  • Maintenance payment collection rate below 60% without tracking
  • Visitor management is ad-hoc with no digital records
  • Admin has no analytics — zero visibility into society health
  • Event/facility booking causes conflicts & double bookings

🟢 The Solution

  • Unified digital platform for all society operations
  • Real-time complaint tracking with status updates & image uploads
  • Integrated Razorpay payments with instant receipt generation
  • Digital visitor log with entry/exit timestamps
  • Admin dashboard with revenue charts, complaint pie charts & analytics
  • Calendar-based facility booking with conflict detection

Website Flow

Complete User Journey

🌐 Public Pages Flow

1
Landing Page — Hero, features showcase, testimonials, stats (500+ societies, 50K+ residents), CTA
↓
2
Register Page — Name, email, password, flat number + Google OAuth sign-up
↓
3
Login Page — Email/password or Google OAuth → JWT token set as cookie
↓
4
Middleware Check — JWT verified → role extracted → redirect to Dashboard or Admin

🔐 Authentication Flow

Email/Password Login:

POST /api/auth/login → validates credentials via bcryptjs → generates JWT with jose → sets flatflow_token cookie → redirects based on role

Google OAuth:

GET /api/auth/google → redirects to Google → callback verifies token → finds/creates user → generates JWT → sets cookie

Middleware Protection:

Every non-public route checks JWT → extracts userId, role, email → injects into headers → admin routes require role="admin"

Security & Infrastructure

Security Architecture

🔒 Authentication & Authorization

  • JWT Tokens — Signed with jose library, stored as HTTP-only cookies
  • Password Hashing — bcryptjs with salt rounds for secure storage
  • Google OAuth 2.0 — Federated identity via Google
  • Middleware Guard — Every request verified at edge before reaching API
  • Role-Based Access — Admin routes blocked for residents via middleware
  • Token Expiry — Automatic redirect to login on expired tokens

🏗️ Infrastructure

  • Deployment — Vercel (serverless, edge functions)
  • Database — MongoDB Atlas (cloud-hosted, encrypted)
  • File Storage — UploadThing CDN for images
  • Payment Security — Razorpay signature verification
  • Environment Variables — Secrets stored in .env (not committed)
  • CORS & Headers — Secure headers via Next.js middleware
